# 🔓 Видалення прив'язки до Steam ID - Інструкція

## Що змінилося

### ✅ До змін:

- Обов'язково потрібен STEAM_USER_ID у `.env.local`
- Завжди показується бічна панель з бібліотекою ігор
- Помилка, якщо немає авторизації

### ✅ Після змін:

- STEAM_USER_ID тепер **опціональний**
- Два режими роботи:
    - **З авторизацією** - повна функціональність + бібліотека
    - **Без авторизації** - тільки глобальний пошук ігор

## 🧪 Як протестувати

### Тест 1: Режим БЕЗ авторизації

1. **Закоментуй STEAM_USER_ID в `.env.local`:**

    ```env
    STEAM_API_KEY=your_key
    # STEAM_USER_ID=your_id  <-- закоментуй цей рядок
    ```

2. **Перезапусти dev server:**

    ```bash
    npm run dev
    ```

3. **Очікувана поведінка:**
    - ✅ Landing page відкривається нормально
    - ✅ Пошук ігор працює
    - ✅ Після вибору гри - відкривається dashboard БЕЗ sidebar
    - ✅ Зверху з'являється панель з кнопками:
        - "← Назад" (зліва)
        - "TrophyHunter" (по центру)
        - "Увійти" з іконкою Steam (справа)
    - ✅ Показуються глобальні досягнення вибраної гри
    - ✅ Немає помилок у консолі

### Тест 2: Режим З авторизацією

1. **Розкоментуй STEAM_USER_ID в `.env.local`:**

    ```env
    STEAM_API_KEY=your_key
    STEAM_USER_ID=your_id  <-- розкоментуй
    ```

2. **Перезапусти dev server:**

    ```bash
    npm run dev
    ```

3. **Очікувана поведінка:**
    - ✅ Landing page показує кнопку "Увійти через Steam" знизу зліва
    - ✅ Після натискання "Увійти" - відкривається dashboard З sidebar
    - ✅ Sidebar показує:
        - Логотип TrophyHunter зверху
        - Список "Моя бібліотека" з іграми
        - Профіль користувача знизу
    - ✅ Можна вибирати ігри з бібліотеки
    - ✅ Показуються особисті досягнення (виконані/не виконані)

## 📝 Технічні деталі

### API Routes

- `GET /api/steam` - повертає `{authenticated: false}` якщо немає USER_ID
- `GET /api/steam/games` - повертає `{authenticated: false}` якщо немає USER_ID
- Обидва ендпоінти тепер не падають з помилкою 500

### Hook: useSteamProfile

Тепер повертає додаткове поле:

```typescript
interface UseSteamProfileReturn {
    profile: SteamProfile | null;
    games: Game[];
    isLoading: boolean;
    error: string | null;
    isAuthenticated: boolean; // НОВЕ!
}
```

### UI Logic

```typescript
{isAuthenticated && <GameSidebar ... />}  // Показуємо тільки якщо є профіль
{!isAuthenticated && <TopBar ... />}       // Показуємо тільки якщо немає профілю
```

## 🚀 Deployment checklist

Перед деплоєм на Vercel:

1. ✅ Додай STEAM_API_KEY до environment variables
2. ⚠️ НЕ додавай STEAM_USER_ID (якщо хочеш публічний доступ)
3. ✅ Протестуй без USER_ID локально
4. ✅ Переконайся, що немає console.error для неавторизованих

## 💡 Примітка для розробника

Якщо в майбутньому зробиш справжню Steam OAuth авторизацію:

- Замість STEAM_USER_ID з .env використовуй session/token
- API routes мають брати steamid з сесії користувача
- Hook `useSteamProfile` зможе передавати steamid динамічно
